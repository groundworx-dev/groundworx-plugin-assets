Groundworx Cards & Sections – Changelog

2.1.0

New

  • New Info Card block with full layout controls and default variation
  • New Video Modal Card block — triggers a video dialog from a card
  • New Video Dialog block for embedding videos inside dialogs
  • New Dialog block system: Dialog, Dialog Content Wrapper, Dialog Header, Dialog Content
  • New gwx_dialog post type with topic and tag taxonomies for reusable dialogs
  • Dialog Manager admin UI under the Groundworx menu (canvas + sidebar editor)
  • Dialog interactivity module rendered in wp_footer for site-wide dialog support

Accessibility

  • Improved ADA compliance and keyboard navigation across card and dialog blocks

Security

  • Additional output escaping and sanitization tweaks

Fixed

  • Video Card render and editor adjustments

2.0.1

  • Added bidirectional transforms between Video Card and core/video blocks
  • Fixed video playback button state not updating when paused by external carousel controls
  • Fixed video playback button showing wrong state on initial load when inside an inactive carousel slide

2.0.0

New

  • New Video Card block with video background and built-in playback controls
  • Video playback control system (play/pause/replay + progress ring) shared across all video-capable blocks
  • Video poster image support with auto-generation from video frames
  • Featured image support for Card, Card Overlay, and Card Reveal backgrounds
  • Automatic contrast detection for Media Card controls based on background brightness
  • Interactivity API-based responsive layout for Media Content Split
  • Video settings panel in editor (autoplay, loop, muted, play inline)
  • Backdrop blur setting for Curtain Raiser overlay
  • Settings page under Groundworx admin menu

Accessibility

  • prefers-reduced-motion support across all blocks — disables CSS transitions, GSAP animations, and video autoplay
  • prefers-reduced-transparency support for Curtain Raiser — overlay becomes opaque, backdrop blur removed
  • Card Reveal falls back to touch layout (content always visible) when reduced motion is enabled
  • Curtain Raiser hides interactive layer when reduced motion is enabled (shows primary background only)
  • Keyboard focus trigger for Card Reveal and Media Card — reveal layers open on tab focus
  • Decorative elements (overlay, blur, background video/image) marked with role="presentation"

Improved

  • Consolidated build system with central entry points for editor and frontend
  • Animations rewritten to use Interactivity API (replaces individual GSAP animation files)
  • Shared components extracted from external package to local plugin
  • Media Content Split responsive behavior driven by Interactivity API instead of CSS media queries
  • Block registration updated to metadata object pattern for reliable deprecated entry support
  • Admin menu restructured with shared Licenses page across all Groundworx plugins
  • Improved internationalization across all user-facing strings

Fixed

  • Block deprecation handlers for Curtain Raiser, Media Card, and Media Section
  • Curtain Raiser overlay no longer appears on blocks saved without explicit opacity
  • Media Content Split responsive class switching in editor matches frontend behavior
  • Text domain corrected to groundworx-cards-sections across all files

Security

  • Improved output escaping and input sanitization across all blocks
  • Added direct access protection to all PHP files

1.0.1

  • Fixed text domain to use groundworx-cards-sections consistently across all files
  • Added Text Domain header to plugin file

1.0.0

  • Initial release
  • Card, Card Overlay, Card Reveal, and Media Card blocks
  • Media Content Split, Media Section, and Curtain Raiser blocks
  • Featured image support for Query Loop integration
